# BentoTUI Architecture

BentoTUI uses a copy-and-own delivery model: UI source is copied into your codebase, while a small set of stable packages stays imported as normal Go modules.

## Delivery Model

| Layer | Purpose |
|---|---|
| `registry/components/` | Source copied by `bento add` into your project |
| `bentos/` | Runnable full-screen patterns you copy and adapt |
| module packages | Stable imports for theme and layout plumbing |

```text
registry/components/   <- copied to your app by bento add
bentos/                <- complete screen templates
```

## Primitive Policy

BentoTUI 1.0 does not wrap primitives that are already strong in Bubbles unless there is clear composition value.

- Use `charm.land/bubbles/v2/spinner` directly for loading states.
- Prefer direct Bubble primitives for low-level behavior.
- Use Bento components where composition and layout structure are the primary value.

## Core Packages

| Package | Import | Role |
|---|---|---|
| `theme` | `github.com/cloudboy-jh/bentotui/theme` | Global theme store and preset selection |
| `styles` | `github.com/cloudboy-jh/bentotui/styles` | Theme-aware Lip Gloss style mapping |
| `layout` | `github.com/cloudboy-jh/bentotui/layout` | Horizontal and vertical split helpers |

## Surface Rendering

`surface` wraps Ultraviolet cell rendering to ensure deterministic full-frame painting and avoid ANSI whitespace-reset bleed.

```go
// Set once at startup
theme.SetTheme("tokyo-night")

// Read current theme during each View render
t := theme.CurrentTheme()
```

## CLI Boundary

```bash
# Install the CLI
go install github.com/cloudboy-jh/bentotui/cmd/bento@latest

# Copy source components
bento add panel bar surface

# Inspect what is available
bento list
```
